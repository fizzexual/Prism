# Adding and working with 3D models

## Using models in the editor

1. Drop a **3D Object** from the palette onto the canvas (it shows a placeholder
   cube until a model is loaded).
2. Select it and open the **3D Model** group in the inspector.
3. Click **Upload model** and choose a `.glb`, `.gltf`, `.obj`, or `.fbx` file.

When a project is saved on the server, uploads go to `POST /api/projects/:id/assets`
and are served from `/uploads/...`. With no server/database available, the file is
embedded as an inline data URL instead, so 3D still works fully offline.

Transform (gizmos + numeric fields), material override, lighting, environment, and
camera are all editable in the inspector. Drag the on-canvas gizmo to translate /
rotate / scale (switch modes in the **Transform** group).

## Supported formats

Loaders live in
[`client/src/three/loaderRegistry.js`](../client/src/three/loaderRegistry.js):

| Format | Extension | Notes |
|---|---|---|
| glTF | `.gltf`, `.glb` | Primary, fully supported |
| OBJ | `.obj` | Supported (no separate `.mtl` materials — use the material override) |
| FBX | `.fbx` | Supported best-effort |

## Adding a new format

Each loader is an async function that resolves a `THREE.Object3D`. Loaders are
dynamically imported so they code-split. To add, say, **STL**:

```js
// in loaderRegistry.js
async function loadSTL(url) {
  const { STLLoader } = await import('three/examples/jsm/loaders/STLLoader.js');
  const THREE = await import('three');
  return new Promise((resolve, reject) => {
    new STLLoader().load(
      url,
      (geometry) => {
        // STL gives geometry, not an Object3D — wrap it in a mesh.
        const mesh = new THREE.Mesh(geometry, new THREE.MeshStandardMaterial({ color: '#cccccc' }));
        resolve(mesh);
      },
      undefined,
      reject,
    );
  });
}

const loaders = {
  gltf: loadGLTF, glb: loadGLTF, obj: loadOBJ, fbx: loadFBX,
  stl: loadSTL, // 👈 add it here
};
```

The new extension automatically becomes accepted by the upload control
(`supportedExtensions` drives the file picker) and by exported scenes — but you
must also teach the **exporter** about it.

## Export

Exported 3D scenes are regenerated by
[`client/src/export/threeGen.js`](../client/src/export/threeGen.js), which emits a
standalone `scene.js` importing three.js from a CDN. Its `load()` switch mirrors
the loader registry — add a branch there for any new format:

```js
else if (f === 'stl') { /* import STLLoader from 'three/addons/loaders/STLLoader.js' and wrap in a mesh */ }
```

Models referenced by server URLs are fetched and bundled into the export's
`assets/` folder; inline data-URL models are embedded directly.

## Limits & tips

- Server upload cap is 50 MB per file (`server/src/routes/projects.js`).
- Each 3D object is an independent WebGL context. Browsers allow ~16 at once, so
  prefer a handful of 3D objects per page.
- Exported 3D requires serving the folder over http(s) (ES-module imports don't
  work from `file://`). See the exported `README.txt`.
